# GoFlow: Высокопроизводительная система сбора событий

GoFlow — это распределенная высоконагруженная система на языке **Go**, предназначенная для приема, обработки и хранения огромных массивов данных в реальном времени.

## 🚀 Показатели производительности
> **Результат тестирования:** Система стабильно выдерживает нагрузку более **71,000 RPS** (запросов в секунду) на одном узле с **нулевыми потерями данных**.

* **Скорость приема (Ingestion):** ~71k зап/сек (Gateway → Kafka)
* **Скорость обработки (Processing):** ~15k-20k зап/сек (Worker → ClickHouse)
* **Отказоустойчивость:** 100% сохранность данных при экстремальных пиках нагрузки. Архитектура использует Kafka как «шоковый абсорбер», позволяя буферизировать миллионы событий во время «бури» трафика.

## 🏗 Архитектура и дизайн
Система построена на конвейерной асинхронной обработке для эффективного управления нагрузкой:
* **Gateway Service**: Сверхбыстрая точка входа на Go. Приоритезирует скорость приема данных, отвечая `202 Accepted` за ~10мс.
* **Apache Kafka**: Надежная шина данных (настроено 10 партиций). Изолирует процесс приема от процесса записи, предотвращая перегрузку базы данных.
* **Worker Service**: Группа воркеров с логикой **батчинга**. Выполняет оптимизированную массовую вставку (до 7,000 строк за раз) в ClickHouse для минимизации нагрузки на CPU и диск.
* **ClickHouse**: OLAP-база данных, обеспечивающая мгновенный доступ к аналитике по миллионам записей.

## 🛠 Технологический стек
- **Язык**: Go (Golang)
- **Брокер сообщений**: Apache Kafka
- **Хранилище**: ClickHouse
- **Оркестрация**: Kubernetes (K8s) и Docker Compose
- **Мониторинг**: Prometheus и Grafana
- **Нагрузочное тестирование**: k6

## 📊 Мониторинг и метрики
Интегрированный стек мониторинга позволяет в реальном времени отслеживать:
- **Пропускную способность**: Входящий RPS vs Скорость обработки.
- **Kafka Lag**: Состояние очереди сообщений во время стресс-тестов.
- **Здоровье системы**: Потребление ресурсов всеми компонентами сервиса.

## 📦 Инфраструктура
Проект содержит готовые конфигурации как для **Docker Compose**, так и для **Kubernetes (K8s)**, что позволяет разворачивать систему в различных окружениях.
